## 🎨 My differential

Before focusing on QA, I spent years exploring **design, visual creation, and digital interfaces**.

That background helps me look at software from different angles:

* Does it work?
* Is it clear?
* Is it consistent?
* Is the user flow intuitive?
* Could this interface confuse someone?
* Is the experience aligned with what the user expects?

For me, quality is not only about finding bugs.
It's also about understanding how people interact with software.
